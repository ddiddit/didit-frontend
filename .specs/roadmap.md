## 핵심 전략

회고 앱이 죽는 패턴: **"쓰긴 쓰는데 다시 안 본다 → 가치를 못 느낀다 → 안 쓴다."**
우선순위는 **루프를 닫는 기능 > 모아보는 기능 > 꾸미는 기능**.

한 줄 스토리:

> **회고를 더 좋게 만들고(개선) → 팔 물건을 만들고(Pro 기능) → 돈을 받는다(결제)**

**핵심: 결제는 맨 마지막.** 게이팅할 Pro 가치가 먼저 존재해야 구독이 팔린다. 빈약한 Pro에 결제부터 붙이면 전환이 0이다.

기능 의존성 때문에 순서는 사실상 강제됨:

```
회고 2.0      →   Pro 강화          →   결제          →   광고
(결과·액션·       (리포트·대시보드·      (게이팅할          (freemium
 맥락 데이터       여정 = 팔 물건이        Pro가 존재해야     한도가 있어야
 를 생성)          쌓인 데이터로 완성)      페이월이 의미)      리워드광고 의미)
```

## 배포·플랫폼 전략 (앱 우선)

**제품 = 네이티브 앱(Capacitor).** 웹은 제품 표면이 아니라 입구(랜딩)로만 쓴다. 이 결정으로 "웹 결제를 막아야 하나" 같은 고민이 통째로 사라짐 — 결제는 앱 IAP 하나로 단순화.

```
didit.ai.kr          →  랜딩 페이지 (공개: 소개 + 다운로드/베타 신청)
app.didit.ai.kr      →  실제 앱 = 운영 빌드 + 운영 API
                        · 버셀 프로덕션이라 보호 게이트 없음 → QA 그냥 접속 ✅
                        · 로그인 미들웨어로 막혀 계정 없는 일반인은 못 씀 → 자연 차단
(vercel 프리뷰 URL)   →  내부 코드확인용 (Deployment Protection으로 팀만)
```

**QA = 하이브리드 (운영 환경 기준):**
- ① **운영 앱 도메인**(`app.didit.ai.kr`)에서 UI·로직 QA — 프로덕션 빌드 + 운영 API. 로그인 게이트로 일반인 자연 차단.
- ② **TestFlight(iOS) / 구글플레이 내부테스트** — 결제(IAP 샌드박스)·푸시·마이크(STT)·파일 등 **네이티브 실동작** QA.

> **왜 운영에서 QA?** 버셀 **프리뷰 배포는 Deployment Protection(팀 로그인)** 때문에 외부 QA가 못 들어옴(= "스테이징 못 들어온다"의 원인). **프로덕션 배포는 공개**라 운영 앱 도메인에서 QA하면 접근 문제 자체가 없음. + 운영 빌드/운영 API로 테스트하니 실제와 동일.

**플랫폼 분기 (같은 Nuxt 코드가 웹/앱 양쪽에서 돎):**
- `Capacitor.isNativePlatform()` 로 감지 → **구독 결제 UI는 네이티브일 때만** 노출 (웹은 IAP 불가).
- Pro 권한은 **계정 기반**(백엔드가 영수증 검증 후 계정에 Pro 표시) → 웹·앱 어디서 로그인해도 Pro 기능 열림.
- IAP 구현은 **RevenueCat**(Capacitor 플러그인) 추천 — 영수증 검증·갱신·계정 권한 동기화 자동, 1인 개발 시간 절약.

- **win:** 질문응답 → "진짜 대화" 전환. 핵심 가치.
- 비고: 프론트보다 **백엔드 프롬프트 비중**이 큼 (채팅 UI는 `start.vue`에 이미 존재).

## 업데이트 2 — "Pro 강화" (팔 물건 만들기)

**결제 전에 먼저.** 데이터가 쌓일수록 가치가 커지는 lock-in 기능 묶음 = 곧 구독의 간판 상품.
업데이트 1에서 만든 구조화 결과·액션·프로젝트 맥락 데이터를 재료로 완성된다.

- **AI 주간/월간 리포트** ⭐️: 흩어진 회고를 AI가 종합 ("이번 주 *전달 방식* 고민 3번", "반복 막힘: 일정 관리"). **푸시 재방문 트리거 최강** ("이번 주 리포트 도착").
- **액션 추적 ③ 대시보드**: 완료율, 누적 완료 N개, 성장 지표 통계 (캡처·루프백은 업데이트 1에서 이미 완료).
- **프로젝트 여정 뷰**: 프로젝트별 회고를 시간순으로 — 문제 → 시도 → 배운 점 흐름 시각화.

> 이 셋이 곧 다음 업데이트의 페이월 콘텐츠. 무료로 맛보기만 주고 풀버전은 Pro로 게이팅한다.

## 업데이트 3 — "수익화" (BM 가동)

업데이트 1·2가 "팔 물건"을 다 완성한 직후의 조용한 BM 업데이트.
**회고 UX 작업과 섞지 않음** — 결제는 인프라 작업이라 섞으면 스프린트가 길어지고 "짧게짧게"가 깨짐.

> **📌 결제 수단 — 사업자등록증 없이 가는 길 (확정 방향)**
> - **웹 직접 결제(국내 PG: 포트원·토스페이먼츠 등)는 사업자등록증 + 통신판매업 신고 필수** → 사업자 없이는 불가.
> - **사업자 없이 가능한 경로 = 앱스토어/구글플레이 IAP(인앱결제).** 스토어가 판매자(merchant of record)라 결제·부가세·환불 대행, **개발자는 개인 계정으로 등록 가능.** (디지털 구독은 어차피 스토어 정책상 IAP 의무라 PG 못 씀)
> - **수수료 15~30%** 차감 후 개인 통장 정산. 단점: **네이티브 앱(Capacitor 패키징)이 먼저 나와야 함** — 순수 웹에선 이 경로 불가.
> - **세금:** 정산금도 과세 대상이고 구독 지속 판매는 사업 활동이라 결국 사업자등록 필요. 단 홈택스에서 무료·10분(간이과세 가능). → **초기엔 개인 IAP로 시작 → 매출 나면 등록**이 현실적.
> - **⇒ 타이밍 결정: 수익화(이 업데이트)를 Capacitor 네이티브 출시 시점에 맞춰 IAP로 붙인다.** 웹 PG·사업자 벽을 우회하고, 어차피 할 패키징과 타이밍이 자연스럽게 맞음.
> - ⚠️ 본 메모는 일반 실무 가이드. 진행 전 홈택스/세무사 확인 권장. (규정 변동 가능)

### 스프린트 — 결제 (크기: M)
LLM API 호출이 진짜 원가 → **AI 사용량 기반 프리미엄(구독)**.

**게이팅 원칙: 횟수가 아니라 "깊이"로 막는다.**
회고 *작성*(핵심 행동)을 횟수로 막으면 안 됨. 미션·배지(주 3회, 7일 내 3회, 데일리 스트릭)가 "주 1회 이상"을 요구 → 프리 한도를 주 1회로 잡으면 무료 유저가 핵심 미션을 달성 못 하는 자기모순. 그래서 **작성은 넉넉히(하루 1회) 열고, 돈은 심화·리포트·무제한에서 받는다.**

| | Free | Pro(구독) |
|---|---|---|
| 회고 작성 | **하루 1회** | 무제한 |
| 회고 결과 | 기본 구조화 | **심화 인사이트 + 서술형 자세히보기** |
| 주간/월간 AI 리포트 | 맛보기 | **풀버전** |
| 액션 대시보드 | ✕ | **O** |
| 프로젝트 여정 뷰 | ✕ | **O** |
| 프로젝트 개수 | 1~2개 | 무제한 |
| STT 음성입력 | 제한 | 무제한 |

- **"하루 1회"가 스윗 스팟:** 모든 미션·배지가 하루 1회로 달성 가능(주 3회 ✓, 7일 내 3회 ✓, 데일리 스트릭 ✓) → 게임 시스템 안 깨짐. LLM 비용은 월 ~30회로 캡.

**가격 정책 (단일 Pro 티어):**

| 플랜 | 가격 | 월 환산 | 비고 |
|---|---|---|---|
| 월간 | **₩7,900/월** | ₩7,900 | 한국 구독앱 심리적 천장(₩9,900) 밑 |
| 연간 ⭐️ | **₩59,000/년** | ₩4,900 | ~38% 할인 → 연간 유도(현금흐름·리텐션 ↑) |

- **얼리버드:** 초기 구독자 한정 혜택(예: 연 ₩39,000 고정)으로 시드 확보 + 입소문.
- **티어 쪼개지 않음:** Free / Pro 단일 구조. Basic/Plus 분리는 관리만 복잡.
- **가격은 출시 후 조정:** ₩7,900으로 시작 → 전환율·원가 데이터 보고 조정. 낮게 시작해 올리면 반발 크므로, 여유 있게 시작해 할인으로 푼다.

**⚠️ 전제 — 모델 비용 (무제한 흑자 조건):**
"무제한 + ₩7,900"은 대화에 어떤 모델을 쓰느냐에 따라 흑자/적자가 갈림(헤비유저 월 30회 기준 효율모델 ~₩5,000 vs 고급모델 ~₩50,000+).
→ **대화(고빈도)는 효율 모델(Haiku급), AI 리포트(저빈도·고가치)만 고급 모델.** 이 조합이라야 무제한이 성립. (회고는 *진짜 대화*라 자연적으로 월 30회 넘기 어려워 무제한 리스크 낮음.)

### 스프린트 — 광고 (리워드형, 크기: S)
무료 한도에서 막혔을 때 **하드월 대신 선택지**를 주는 freemium 압박 밸브.

```
하루 1회 다 씀  →  더 쓰고 싶다
                    ├── 📺 광고 보고 +1회
                    └── ⭐️ Pro로 무제한
```

| 구분 | 한도 | 비고 |
|---|---|---|
| Free | 회고 하루 1회 | 기본 결과만 |
| + 리워드 광고 | +1회씩, **하루 최대 3회까지** | 횟수만 풀림 |
| Pro | 무제한 + 심화·리포트·대시보드·여정·무제한 프로젝트 | **광고 0** |

- **광고가 푸는 건 "횟수"뿐, "깊이"는 절대 안 풀림** → 심화·리포트·대시보드·여정은 오직 Pro. (**광고 = 양, Pro = 질**)
- 리워드형만. **회고 챗봇 플로우·결과 화면 중간엔 절대 금지**(몰입 공간). Pro 유저는 광고 0.
- 일반 배너는 수만 DAU 이후에나 (구독이 본진, 광고는 보조).
