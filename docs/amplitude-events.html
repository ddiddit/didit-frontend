Amplitude 이벤트 정의서 (Tracking Plan)

didit 프론트엔드의 앰플리튜드 이벤트 명세. 새 이벤트를 추가하거나 프로퍼티를 바꿀 때 이 문서를 함께 갱신한다.

기본 원칙

  • 이벤트명: object_action 형태의 snake_case (예: retrospect_completed)
  • 프로퍼티명: snake_case
  • 사전 등록 불필요: track() 호출 즉시 앰플리튜드에 수집된다. 이 문서는 거버넌스/일관성용.
  • 수집 환경: 개발 환경(dev)에서는 전송 안 됨. 프로덕션에서 API Key 필요.

유저 식별 (User Identity)

시점동작위치
로그인/회원가입identify() → JWT의 사용자 ID로 식별login.vue, kakao/callback.vue
탈퇴reset()my/account/withdraw.vue
  • 리텐션·DAU·MAU·stickiness는 위 식별 + 이벤트 타임스탬프로 앰플리튜드가 자동 계산한다(별도 이벤트 불필요).
  • SDK가 userId를 자체 저장하므로 세션이 바뀌어도(토큰 자동로그인) 식별이 유지된다.
  • 앱 부팅 시 토큰이 있으면 setUserId를 호출해 식별 누락 엣지 케이스를 방지. (반영 완료)

프로퍼티 설명

프로퍼티의미
provider소셜 로그인 수단 (kakao / google / apple)

question_no몇 번째 질문에 답했는지
