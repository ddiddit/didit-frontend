지정된 `.specs/` 파일과 현재 코드베이스를 비교 검토하세요.

## 검토 항목

1. **인수 기준 충족도** — 각 AC별로 ✅ / ❌ / ⚠️ 표시
2. **비목표 위반** — 만들면 안 되는 걸 만들었는지
3. **CLAUDE.md 규칙 준수** — 코딩 컨벤션, 보안 규칙 위반 여부
4. **코드 품질**:
   - 보안 (인젝션, XSS, 권한 우회, 민감 정보 노출)
   - 성능 (불필요한 연산, 비효율 쿼리, N+1 문제)
   - 가독성 (변수명, 함수 분리, 주석)
   - 유지보수성 (SRP, DRY, 결합도)
5. **테스트 커버리지** — 인수 기준당 테스트 있는지
6. **누락된 엣지 케이스**

## 주의

- 인수 기준에 명시되지 않은 추가 기능은 비목표 위반
- 트레이드오프가 있는 결정은 "왜 이렇게 했는지" 추측하지 말고 질문하기
- CLAUDE.md의 절대 금지 사항 위반은 무조건 CRITICAL
- 작은 스타일 이슈는 LOW로 분류
